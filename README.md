# Проект: Task Manager

## Описание проекта
Веб-приложение для управления личными задачами пользователей. Обеспечивает безопасное разделение прав доступа (каждый пользователь работает строго со своими данными), поддерживает регистрацию, авторизацию, полноценный CRUD, а также совместный поиск, фильтрацию и пагинацию.

### Краткое описание функционала:
* **Авторизация:** Вход, выход и регистрация пользователей на базе встроенной Session Authentication Django.
* **Управление задачами:** Создание, просмотр деталей, изменение и удаление личных задач.
* **Поиск и фильтрация:** Поиск по названию совместно с фильтрацией по статусам («Новая», «В работе», «Выполнена»).
* **Пагинация:** Ограничение вывода до 10 задач на одну страницу.

## Используемые технологии
* **Python** 3.12+
* **Django** 5+ (с использованием Class Based Views)
* **PostgreSQL** (основная реляционная СУБД)
* **Bootstrap** (фронтенд-сетка и стилизация компонентов)

---

## Структура проекта
Краткое описание ключевых директорий:
* `core/` — Главная конфигурационная папка проекта (настройки `settings.py`, корневой `urls.py`).
* `tasks/` — Изолированное приложение для управления задачами (модели, логика CRUD, формы, админка).
* `users/` — Изолированное приложение для кастомной регистрации пользователей.
* `templates/` — Централизованные HTML-шаблоны страниц и форм авторизации.
* `.env.example` — Шаблон конфигурационного файла для безопасной настройки переменных окружения.

---

## Требования к коду (Соблюденные стандарты)
* **Соблюдение PEP8:** Код отформатирован в соответствии со стандартами стиля Python.
* **Class Based Views:** Вся бизнес-логика и CRUD-операции реализованы строго на классах (дженериках Django).
* **Безопасность данных:** Секретные ключи приложения и параметры подключения к БД не хранятся в коде, а считываются из `.env`.
* **Архитектура:** Логика управления задачами вынесена в отдельное приложение `tasks`.
* **Разграничение прав:** Доступ анонимов заблокирован через `LoginRequiredMixin`, а просмотр/изменение чужих задач пресекается на уровне базы данных с помощью переопределения метода `get_queryset()`.
